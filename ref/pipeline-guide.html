개발
1
이슈 & 전체 댓글 로드
플랜·피드백 댓글 모두 시간순으로 파악해 개발에 반영
2
GitHub repo 생성 (신규 프로젝트)
repo 없으면 자동 생성(private) → Workflow permissions: write 자동 설정 → clone
3
기술 스택 자동 판단
기존 파일 기반 감지 / 신규는 이슈 내용 분석 → nextjs / express / python
4
포트 고정 할당 (3001~3010)
프로젝트별 고정 포트를 _port_map.json에 영구 저장
5
Feature Branch 생성
feature/{ISSUE_ID}-{kebab-title} 규칙으로 생성
6
코드 개발
스택에 맞는 프로젝트 초기화 + 이슈·댓글 내용 기반으로 실제 기능 구현
7
개발 서버 구동 (자동 오류 수정 루프)
서버가 HTTP 응답할 때까지 오류 분석 → 코드 수정 → 재시도 무한 반복. 해결 불가 시 Slack 알림 후 중단.
8
Commit & Push → PR 생성
feature branch에 push 후 PR 자동 생성
테스트
10
Playwright E2E 테스트 (프론트엔드) + 자동수정 루프
nextjs/react 스택에서만 실행. 테스트 파일 없으면 앱 구조 분석 후 자동 생성. 실패 시 코드 수정 → 서버 재기동 → 재시도 최대 5회.
11
백엔드 테스트 (Jest / pytest) + 자동수정 루프
테스트 파일 있으면 실행. 없으면 주요 API 엔드포인트 curl 점검. 실패 시 코드 수정 → 재시도 최대 5회.
보안
12
SAST + SCA + DAST 보안 점검
• SAST: Semgrep (OWASP Top 10, CWE Top 25, Secrets 룰셋)
• SCA: npm audit / pip-audit → critical/high 시 npm audit fix 자동 실행
• DAST: Nuclei (정적 바이너리, owasp·cve·exposure·misconfig 템플릿) — Docker 불필요
• SAST HIGH 취약점 → 파일:라인 기반 자동수정 → 재스캔 (최대 3회)

✅ Squash Merge부터 배포 파일 생성까지 자동입니다. 이후 NAS에서 직접 실행하면 됩니다.
1
PR 상태 확인 후 Squash Merge
머지 불가 시 Slack 알림 후 중단
2
Feature Branch 삭제
GitHub API로 원격 브랜치 자동 삭제

Dockerfile + docker-compose.yml + .env.example 생성
• 프로젝트 유형 자동 감지 (nextjs / nodejs / python / golang) → 멀티스테이지 Dockerfile
• docker-compose.yml: 로컬 빌드 (build: .), 포트는 ${HOST_PORT} 환경변수로 수동 지정
• .env.example: HOST_PORT= 안내 포함
• .dockerignore 자동 생성
→ main 브랜치에 commit & push
4
Linear 댓글 + Done + Slack 알림
NAS 수동 배포 가이드 포함. repo 링크·Dockerfile 위치 안내. Slack 알림에 git clone → .env 설정 → docker compose up -d --build 명령어 코드블록 포함.
ℹ️ 이후 NAS에서 repo를 clone하고 .env에 HOST_PORT를 지정한 뒤 docker compose up -d --build를 실행합니다.

Linear 워크스페이스에 아래 상태들이 존재해야 합니다.

상태명유형담당

GitHub PAT 갱신
만료 시
▼
⚠️ PAT가 만료되면 GitHub API 호출이 모두 실패합니다. .env의 GITHUB_TOKEN 값만 교체하면 됩니다.

갱신 절차

단계내용
1GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2사용 중인 토큰 클릭 → Regenerate token (권한은 그대로 유지됨)
3생성된 새 토큰 복사
4서버에서 .env 수정: GITHUB_TOKEN=ghp_새토큰값
5cron 방식은 다음 실행 시 자동 반영 / webhook-server.js 사용 시 프로세스 재시작

필요 권한: repo, pull_requests, workflow

NAS 배포 — 프로젝트 최초 실행
프로젝트당 1회
▼
ℹ️ 첫 Reviewed 머지 후 repo의 main 브랜치에 Dockerfile과 docker-compose.yml이 자동으로 생성됩니다.

NAS에서 아래 순서로 실행합니다:

# 최초 1회
git clone https://github.com/your-id/project-name.git
cd project-name
echo 'HOST_PORT=원하는포트번호' > .env
docker compose up -d --build   # Dockerfile로 로컬 빌드 후 실행

이후 업데이트 시:

cd project-name
git pull origin main
docker compose up -d --build
